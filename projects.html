TitanTypes

TitanTypes is a web-based typing test application that allows users to type along to randomly-generated words & quotes, and subsequently gauge their typing speed alongside a variety of other statistics. This web application was written using HTML5, CSS3, JavaScript, supplemented by the jQuery library for additional functionality, and was created by myself alongside two other teammates. Team collaboration and subsequent workflow was organized through the Scrum framework, where core Agile values were incorporated to ensure the success of our team and the subsequent output of a high-quality project.

Who Would Win?

Who Would Win?  is an interactive web application where users can vote on match-ups between their favorite characters from video games, television shows, movies, books, etc. These matchups are randomly-generated, where characters are selected from our Google Firestore database that provides the central database functionality for the application. The overall interface was created utilizing pure HTML5 and CSS3, while Next.js (a React and Node-based framework) was used for providing the application logic and real-time rendering.

TableTop Online

TableTop Online  is an interactive web platform where you can play against individuals in a variety of card games. Currently, this web application features the game of War. Implemented through the use of React, Node.js, and Socket.io, this web application allows users to create accounts that will be registered in a Heroku database. From there, users can view a list of available popular card games, and select the game of their choosing. They can then see available rooms in which they can join, or create their own room. Following connection to a room, they can fully engage in multiplayer action with remote users in the card game of their choosing. This web application was developed with 6 other individuals, where I served as the Scrum Master of the project.

University Database

A small database that holds professor records, student records, and a variety of other entities with their respective attributes. To lay out the framework for this project, my team and I began by designing both an ER Diagram and Relational Model as our schemas in order to help us structure our database entitities and their relations. Once this was complete, we utilized MySQL for our database tables, and populated them with sample data. Next, I worked on the front-end design of the website, utilizing HTML5 and CSS3 to create pages and forms to test user queries. Four queries are accessible, and I subsequently used PHP to link the user's input with our database, in which their requests would be processed and the desired results would be displayed.

Simon

This is a C++ implementation of the game Simon that relies on Windows API functions to change the color of the terminal screen. This game operates essentially how a normal game of Simon would, where colors are randomly generated and the user must recall the pattern and see how far along they can progress. There is an existing high-score system where a user's local scores are saved to a file, so they can attempt to do better than their previous playthrough during each session.
